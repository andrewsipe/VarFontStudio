Left panel — axis tree
H
Axis tag pill is missing
Each axis header in the prototype shows a small monospace teal tag (e.g. opsz, wdth, wght) before the axis name. The screenshot shows only the name. This tag is important — it's the technical identifier users actually type in tooling.
Optical size→opsz · Optical size
H
Stop count badge missing from axis header
The prototype shows a muted numeric badge on the right side of each collapsed axis row (e.g. 13, 3, 7). This is a quick signal for how many stops are configured without expanding. The screenshot omits this entirely.
H
Axis range missing from header row
The prototype shows the min–max range (5 – 1200, 88 – 113) in the axis header. The screenshot shows nothing. This is useful context for understanding the axis before expanding it.
M
Stop rows lack numeric value column
In the prototype each stop row shows a small amber monospace number on the left (e.g. 5, 6, 7…) before the name. The screenshot only shows names. The numeric coordinate is core to the tool's purpose.
M
Axis range label between min and stops is absent
Under each axis header, the prototype shows the range in compact form right below the collapse triangle. The screenshot omits this, leaving the stop list without positional context.
M
"STAT only" toggle not visible
The prototype has a checkbox labeled STAT only below each axis header. This is a core feature flag. The screenshot shows no such toggle.
L
Left indentation on stop rows is shallow
The prototype indents stop rows ~22px under the axis header to create clear visual hierarchy. The screenshot's stops feel more flush with the headers. A deeper indent visually groups the children to their parent axis.
Center panel — instances list
H
Instance key / coordinates column is missing
The prototype's table has a two-column layout: Style name and Key. The Key column shows the raw axis coordinates in monospace (e.g. opsz:5|wdth:88|wght:360). The screenshot has no second column — only the name is shown.
H
Include/exclude checkboxes are absent
Every instance row in the prototype has a leading checkbox that controls whether it's included in the export. The screenshot has no checkboxes. This is a primary interaction — without it the tool can't do its main job.
H
Toolbar above the list is missing actions
The prototype has a toolbar above the instance list with a search field, an "Excluded only" toggle, and column headers (Style name / Key). The screenshot only shows the font filename and an "Edited" badge — no search, no filter toggle.
M
No visual grouping of instances by axis stop
The prototype groups instances with sticky section headers (e.g. "Micro", "Minuscule", "Miniature"…) corresponding to the primary axis stop. The screenshot shows a flat list with no grouping separators.
L
Selected instance highlight not visible
The screenshot shows "Micro SemiCond SemiLight" selected in the inspector but the corresponding list row has no visible selection highlight. The prototype uses a tinted purple background on the selected row that ties the two panels together visually.
Right panel — inspector
H
Naming chain is shown as a table, not a visual chain
The prototype renders the naming chain as pill-shaped nodes connected by arrows (OPSZ → WDTH → WGHT) to convey the ordering relationship. The screenshot uses a plain key–value table. The chain metaphor is doing real communicative work — it shows ordering and could support drag-to-reorder.
M
Plan warnings section lacks visual weight
In the screenshot the warnings are a plain list of small amber text lines. In the prototype they each get an amber tinted alert row with a warning icon and background. Given how many warnings can appear, the distinction between individual warnings is important for scannability.
M
Composed name lacks visual emphasis
The prototype wraps the composed name in a tinted block with a colored left border — it's meant to feel like the "output" of the tool. The screenshot shows it as a plain text field. More visual weight here helps users understand this is the result, not just another input.
L
Coordinates section could use two-column layout
The prototype shows coordinates in a table with axis tag, numeric value, and name side by side. The screenshot stacks them as individual labeled rows. The tag–value–name trio reads better in one horizontal band per axis.
Global / structural
H
Naming chain footer bar is absent
The prototype has a full-width footer bar spanning all three panels showing the naming chain with drag-to-reorder nodes and a live preview of the composed name. This is completely absent from the screenshot. Even a read-only version of this would clarify how the name is assembled.
M
Instance count formula is not shown
The prototype's context bar shows the generation formula (e.g. 13 × 3 × 7 = 273) and instance count. The screenshot shows 13 × 3 × 7 = 273 in the left panel header but it's styled as small secondary text. It deserves more prominence — this number is the key output metric of the tool.
L
Panel dividers are visually lighter than the prototype
The three panels in the prototype have clear 1px dividers. The screenshot's panels bleed into each other a bit — the left and right panel backgrounds seem indistinguishable from the center. Stronger contrast between sidebar and content backgrounds would make the three-column structure read more cleanly on macOS.
